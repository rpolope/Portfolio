    C++ Fluid Solver

    Simulación basada en las ecuaciones de Navier-Stokes. Esta implementación se lleva a cabo en C++ usando la librería GLUT. En ella, se usan los métodos de integración numérica de Gauss-Seidel y Jacobi para llevar a cabo los cálculo de los estados del fluido en cada fotograma, calculando los añadidos de densidad y flujo de fluido, así como la advección mediante interpolación bilineal.

    Simulation based on the Navier-Stokes equations. This implementation is done in C++ using the GLUT library. In it, Gauss-Seidel and Jacobi numerical integration methods are employed to perform the calculation of fluid states in each frame. This includes calculating density and fluid flow additions, as well as advection through bilinear interpolation.

    Procedural Terrain Generation in Unity

    Generación de Terreno Procedimental en Unity

    Este proyecto realizado en mis estudios de máster consiste en una herramienta desarrollada en Unity para la generación infinita de terreno conforme el espectador se desplaza por la escena. Implementa la generación de chunks de terreno utilizando programación en paralelo con JobSystem y Burst Compiler con C#, aprovechando el sistema DOTS de Unity para optimizar el rendimiento del procesador y mejorar la tasa de frames. Esto facilita la implementación de mejoras en los algoritmos de generación, como la inclusión de algoritmos de erosión. La herramienta ofrece varios algoritmos de generación de terreno, como ruido Perlin, Simplex y Voronoi, así como la opción de añadir erosión para aumentar el realismo, mediante la coloración de los vértices de la malla según un gradiente de color personalizable. Además de este gradiente, el script de generación incluye diversos parámetros personalizables que permiten ajustar el terreno generado según las preferencias del usuario.

    This project, carried out during my master's studies, consists of a tool developed in Unity for the infinite generation of terrain as the viewer moves through the scene. It implements terrain chunk generation using parallel programming with JobSystem and Burst Compiler using C#, leveraging Unity's DOTS system to optimize processor performance and improve frame rate. This facilitates the implementation of improvements in generation algorithms, such as the inclusion of erosion algorithms. The tool offers various terrain generation algorithms, such as Perlin noise, Simplex, and Voronoi, as well as the option to add erosion to increase realism by coloring the mesh vertices based on a customizable color gradient. In addition to this gradient, the generation script includes various customizable parameters that allow adjusting the generated terrain according to user preferences.

    Deformable 2D-Mesh (Breakable)

    Malla 2D Deformable (Rompible)

    Esta simulación en Processing es una continuación de la experimentación con las simulaciones de mallas 2D basadas en el modelo masa muelle. En este caso se experimenta con la posibilidad de crear una malla deformable con ruptura si se excede una elongación límite entre nodos contiguos.

    En esta simulación se experimenta con una constante elástica muy baja, por lo que apenas se da lugar a la elongación de la malla. De esta manera, se sobrepasa enseguida el límite de elongación y se produce la fractura de la mesh, dando como resultado que algunos pequeños trozos de tela queden flotando dando la ilusión de que es un material sólido no deformable

    This Processing simulation is a continuation of the experimentation with 2D mesh simulations based on the mass-spring model. In this case, the possibility of creating a deformable mesh with rupture if an elongation limit between contiguous nodes is exceeded is explored.

    This simulation explores a very low elastic constant, resulting in minimal elongation of the mesh. In this way, the elongation limit is quickly exceeded, leading to the fracture of the mesh. As a result, some small pieces of the mesh remain floating, creating the illusion of a non-deformable solid material.

    Soft Bodies

    Cuerpos Blandos

    Position Based Dynamics (PBD)

    En esta simulación de Processing realizada también durante mis estudios de grado de nuevo nos encontramos con una simulación de telas, pero esta vez bajo un modelo distinto. En esta simulación se emplea un modelo basado en un paper de Muller et al. (2007) sobre Position Based Dynamics (PBD). Este enfoque, en lugar de utilizar velocidades, se centra directamente en las posiciones, permitiendo evitar problemas de integración en sistemas basados en fuerzas. Una de las características esenciales que destacan en este modelo es el uso de restricciones para el cálculo de las nuevas posiciones. En el vídeo, se observa que los colores de las uniones entre los nodos de la malla tienen un color entre blanco y rojo, siendo más rojo cuando las distancias entre nodos cumplen menos con las restricciones establecidas.

    In this Processing simulation, also conducted during my undergraduate studies, we encounter a cloth simulation, but this time under a different model. This simulation employs an approach based on a paper by Muller et al. (2007) on Position Based Dynamics (PBD). This approach, instead of relying on velocities, focuses directly on positions, avoiding integration problems in force-based systems. One of the essential features highlighted in this model is the use of constraints for calculating new positions. In the video, you can observe that the colors of the connections between the mesh nodes range from white to red, with a deeper red indicating less compliance with the established constraints.

    Particle Systems

    Sistemas de Partículas

    Escalable Systems

    Sistemas Escalables

    En esta práctica de Simulación, se abordan dos modelos de simulación de colisión de partículas. El modelo basado en velocidades, que comprende la detección de colisiones, la restitución a la posición anterior y el cálculo de la velocidad de salida. Y el modelo basado en fuerzas elásticas (muelles), donde se aplica una fuerza repulsiva si las partículas se acercan demasiado. Además, se introducen estructuras de datos como el método basado en grid y tabla hash para mejorar la eficiencia de la detección de colisiones. Estas estructuras contemplan partículas vecinas, reduciendo el coste computacional de la estrategia de "todas contra todas". Se destaca la visualización de partículas agrupadas en celdas para verificar la precisión de las estructuras de datos.

    In this Simulation practice, two particle collision simulation models are addressed. The velocity-based model involves collision detection, restitution to the previous position, and calculation of the exit velocity. The elastic force-based model (springs) applies a repulsive force if particles get too close. Additionally, data structures such as the grid-based and hash table methods are introduced to enhance collision detection efficiency. These structures consider neighboring particles, reducing the computational cost of the "all-to-all" strategy. The visualization of particles grouped in cells is emphasized to verify the accuracy of the data structures.

    Fire Works

    Fuegos Artificiales

    Esta práctica de simulación aborda la creación de un sistema de partículas para simular un castillo de fuegos artificiales. El objetivo principal es implementar interacciones realistas entre las partículas, considerando fuerzas externas como el viento y la gravedad. Cada partícula se caracteriza por propiedades como color, forma y tiempo de vida. La simulación se divide en el ascenso de la carcasa y la explosión para formar diversas palmeras. La interfaz permite configurar la fuerza y dirección del viento, y se deben diseñar diferentes tipos de palmeras mediante configuraciones de velocidad inicial.

    This simulation practice addresses the creation of a particle system to simulate a fireworks display. The main objective is to implement realistic interactions between the particles, considering external forces such as wind and gravity. Each particle is characterized by properties such as color, shape, and lifetime. The simulation is divided into the ascent of the shell and the explosion to form various palm trees. The interface allows configuring the force and direction of the wind, and different types of palm trees must be designed through initial velocity configurations.

    Waves

    Ondas

    Directional, Radial, Gerstner

    Direccional, Radial, Gerstner

    Esta práctica aborda la simulación de ondas mediante mapas de altura, utilizando tres modelos: onda radial, onda direccional y onda Gerstner. El enfoque se centra en la implementación de estructuras de datos gráficas, representadas por mapas de altura con coordenadas (x, y, z). Se estudian y desarrollan las funciones necesarias para simular cada tipo de onda, destacando la adaptación de la onda Gerstner para evitar crestas artificiales.

    This practice addresses the simulation of waves using height maps, employing three models: radial wave, directional wave, and Gerstner wave. The focus is on the implementation of graphical data structures, represented by height maps with coordinates (x, y, z). The necessary functions to simulate each type of wave are studied and developed, with a particular emphasis on adapting the Gerstner wave to avoid artificial crests.
